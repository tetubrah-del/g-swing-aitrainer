# 🧠 Impact：早期伸展（骨盤が前に出る）判定RAG

##（LLM用・体勢崩れ検知／強制出力ルール込み）

---

## 1️⃣ 適用範囲（必須）

```
本RAGはフェーズ：Impact（インパクト）における
「早期伸展（骨盤がボール側に前に出る／前傾が起きる）」を判定する。

Impact評価では、必ずこの項目をチェックに含める（省略不可）。
```

---

## 2️⃣ 定義

### 早期伸展（Early Extension）【定義】

```
インパクト付近で骨盤がボール方向に突っ込み、
前傾角が起きて「腕・クラブの通り道（スペース）」が潰れる現象。
```

---

## 3️⃣ 観測チェック（1つでも該当で「早期伸展あり」）

LLMはフレームから以下を明示的にチェックせよ。

- インパクト直前〜直後で、骨盤（腰）がボール側に前進している
- 前傾が起き、上体が起き上がる方向に変化している
- 腰と上体の距離が詰まり、腕が体から離れて手元が浮く傾向がある
- その結果として「当たり所」「フェース向き」の安定性が下がりそうな姿勢になっている

---

## 4️⃣ 言い換え辞書（表現ブレ防止）

以下は同義として扱うこと（いずれかに正規化してよい）。

```
早期伸展
骨盤が前に出る
腰が前に出る
前傾が起きる
腰の突っ込み
スペースが潰れる
```

---

## 5️⃣ 出力ルール（絶対遵守）

早期伸展が「あり」の場合、LLMは必ず以下を満たす。

### 5.1 issues への強制記載

```
phases.impact.issues に必ず
「早期伸展」または「骨盤が前に出る」または「前傾が起きる」
を含める（抽象表現のみ禁止）。
```

### 5.2 点数制約（推奨）

```
早期伸展ありの場合：
impact.score は 0〜12 に収める

早期伸展が明確（スペース消失／手元が浮く／体勢崩れが強い）なら：
impact.score は 0〜10 に収める
```

---

## 6️⃣ 禁止事項

```
- 「腰の回転不足」だけで済ませる（早期伸展を見逃す）
- フェース向きの話に逃げる
- 原因をトップやアドレスに転嫁してImpact評価を薄める
```
